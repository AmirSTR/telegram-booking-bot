# 📅 Booking Bot — Система записи к мастерам

Telegram-бот для записи клиентов к мастерам. Внутри одного бота живёт неограниченное количество независимых мастеров, каждый со своим прайсом, расписанием и клиентской базой.

---

## 🏗 Архитектура

```
booking_bot/
├── main.py                  # Точка входа, запуск бота и планировщика
├── config.py                # Конфиг из .env
├── requirements.txt
├── db/
│   └── database.py          # Все SQL-запросы (init, CRUD)
├── handlers/
│   ├── super_admin.py       # Управление мастерами (только для супер-админа)
│   ├── master.py            # Панель мастера
│   └── client.py            # Запись клиентов
├── keyboards/
│   └── keyboards.py         # Все инлайн и реплай клавиатуры
└── utils/
    ├── states.py            # FSM-состояния для всех диалогов
    ├── schedule.py          # Расчёт свободных слотов, форматирование дат
    └── reminders.py         # Планировщик напоминаний (APScheduler)
```

---

## 👥 Роли

| Роль | Как определяется | Доступ |
|------|-----------------|--------|
| **Супер-админ** | `SUPER_ADMIN_ID` в `.env` | Команда `/admin` — добавление/удаление мастеров |
| **Мастер** | Запись в таблице `masters` | Команда `/start` без аргументов — панель мастера |
| **Клиент** | Пришёл по ссылке `?start=master_<ID>` | Меню записи, привязанное к конкретному мастеру |

## 🚀 Установка и запуск

### 1. Клонируй и установи зависимости
```bash
cd booking_bot
pip install -r requirements.txt
```

### 2. Создай `.env`
```bash
cp .env.example .env
```
Заполни:
```
BOT_TOKEN=ваш_токен_от_BotFather
SUPER_ADMIN_ID=ваш_telegram_user_id
```
> Свой ID можно узнать через @userinfobot

### 3. Запусти
```bash
python main.py
```

## 📋 Функциональность

### Супер-админ (`/admin`)
- ➕ Добавить мастера по Telegram ID
- 📋 Просмотреть всех мастеров
- ❌ Удалить мастера

### Мастер (`/start`)
- **💼 Услуги** — добавить/удалить услуги с ценой и длительностью
- **📅 Расписание на сегодня** — все записи на текущий день с контактами клиентов
- **📆 Все записи** — список всех записей с управлением (подтвердить / выполнено / отменить)
- **💰 Статистика** — доход за неделю / месяц / год (только выполненные записи)
- **⏰ Рабочее время** — начало/конец дня и длительность слота
- **🔗 Ссылка для клиентов** — персональная deep-link ссылка
- **👥 Лист ожидания** — уведомить клиентов об открывшихся окнах

### Клиент (через ссылку мастера)
- **📝 Записаться** — выбор услуги → дата (только будни) → свободное время
- **📋 Мои записи** — предстоящие записи
- **❌ Отменить запись** — с уведомлением мастера
- **🔔 Лист ожидания** — встать в очередь если нет свободных слотов

### Напоминания (автоматически)
- ⏰ За **24 часа** — с кнопками "Подтверждаю / Отменяю"
- 🔔 За **2 часа** — повторное напоминание
- Мастер получает уведомление о реакции клиента

## 🔑 Ключевые решения

**Изоляция данных:** каждая запись в БД содержит `admin_id` (Telegram ID мастера). Клиент видит только услуги и слоты своего мастера. Данные разных мастеров никогда не пересекаются.

**Свободные слоты:** при выборе времени система берёт все теоретические слоты рабочего дня и вычитает занятые (с учётом длительности каждой услуги). Занятые слоты просто не отображаются.

**Deep links:** каждый мастер получает уникальную ссылку `?start=master_<ID>`. По ней клиент регистрируется в контуре именно этого мастера и далее всегда работает только в нём.

**FSM + контекст:** мастер-ID сохраняется в FSM storage клиента. Это позволяет восстанавливать принадлежность клиента нужному мастеру без повторных запросов по каждому действию.

## 🗄 Схема базы данных

```sql
masters       — мастера (telegram_id, name, work_start, work_end, slot_duration)
services      — услуги (admin_id, name, price, duration)
clients       — клиенты (admin_id, telegram_id, name, phone)
bookings      — записи (admin_id, client_telegram_id, service_id, date, time, status)
waitlist      — лист ожидания (admin_id, client_telegram_id, service_id)
```

Все таблицы связаны через `admin_id` = `telegram_id` мастера.
